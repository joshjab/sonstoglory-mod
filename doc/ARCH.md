# Sons To Glory — Architecture Decision Record

> This document captures the proposed technical architecture for the modernized sonstoglory.com.
> Decisions marked **[TBD]** require answers to open questions before being finalized.

---

## 1. Overview

The new site is a **statically generated website** — all HTML is generated at build time from Markdown content files. There is no server, no database, and no runtime backend. The output is plain HTML/CSS/JS files hosted on a CDN.

```
Author writes Markdown → git push → CI/CD build → Static HTML deployed to CDN → Users browse site
```

This preserves the spirit of the current site (static, fast, low-maintenance) while adding modern tooling, mobile responsiveness, and new features.

---

## 2. Tech Stack

| Layer | Decision | Alternatives Considered | Rationale |
|---|---|---|---|
| **Static Site Generator** | **Astro** | Hugo, Next.js, Eleventy | Markdown-native, component model, zero JS by default, great content collections API, active ecosystem |
| **Content Format** | **Markdown (.md) with YAML frontmatter** | MDX, JSON, Contentful | Human-editable, version-controlled, portable, no vendor lock-in |
| **Styling** | **Tailwind CSS** | plain CSS, Sass, CSS Modules | Utility-first means no unused CSS, responsive utilities built-in, no naming conventions to maintain |
| **Hosting** | **GitHub Pages** (initial) → small AWS if needed | Netlify, Cloudflare Pages | Zero cost to start; iterate fast; migrate to AWS Lightsail only if GitHub Pages proves limiting |
| **Email / Newsletter** | **ConvertKit/Kit free tier** | Buttondown, Mailchimp, Substack | ~500 subscribers exceeds Buttondown's free tier (100); Kit supports 10,000 free |
| **Search** | **Pagefind** | Algolia DocSearch, Fuse.js | Zero cost, runs at build time, no external service, works entirely offline |
| **Analytics** | **Self-hosted Umami** on home server / AWS | Plausible ($9/mo), Google Analytics | Zero cost; privacy-first; no cookie banner; can run on same small instance as any other services |
| **YouTube** | **RSS feed (build-time fetch)** | YouTube Data API v3 | No API key required, no client-side JS, updates on every deploy |
| **Runtime JS** | **Minimal** | — | Only used for: mobile nav toggle, dark mode toggle, search widget, lazy YouTube embed |

## 4. Content Collections Schema

Defined in `src/content/config.ts` using Astro's built-in Zod validation.

### 4.1 Newsletter

```typescript
newsletters: defineCollection({
  schema: z.object({
    title: z.string(),
    author: z.string().default("Paul Jablonowski"),
    date: z.coerce.date(),
    number: z.number().optional(),
    tags: z.array(z.string()).default([]),
    description: z.string().optional(),
    legacy_url: z.string().optional(),
    pdf: z.string().optional(),
  }),
})
```

### 4.2 Book Chapter

```typescript
book: defineCollection({
  schema: z.object({
    title: z.string(),
    chapter: z.number(),
    description: z.string().optional(),
  }),
})
```

### 4.3 Article

```typescript
articles: defineCollection({
  schema: z.object({
    title: z.string(),
    author: z.string().default("Paul Jablonowski"),
    date: z.coerce.date(),
    tags: z.array(z.string()).default([]),
    description: z.string().optional(),
    legacy_url: z.string().optional(),
  }),
})
```

## 6. Build Pipeline

```
npm run build
  │
  ├── prebuild: node scripts/fetch-youtube-feed.mjs
  │     └── writes src/data/videos.json
  │
  ├── astro build
  │     ├── validates all content collection frontmatter
  │     ├── generates all static pages
  │     └── outputs to dist/
  │
  └── postbuild: npx pagefind --site dist
        └── writes dist/pagefind/ (search index)
```

**Deploy trigger:** Every `git push` to `main` triggers a Netlify build automatically.

**YouTube freshness:** Videos page is only as fresh as the last build. To keep it updated, set up a Netlify scheduled build hook (daily cron). [TBD — configure after initial deploy]

## 7. Design System

### Typography
- **Headings:** Lora (Google Fonts, serif) — warm, authoritative
- **Body:** Source Serif 4 (Google Fonts) — optimized for long-form reading
- **Base size:** `text-lg` (18px) as default body, bumped to `text-xl` (20px) on desktop
- **Max reading width:** `max-w-2xl` (~672px) for article body text

### Color Palette

| Token | Light Mode | Dark Mode | Usage |
|---|---|---|---|
| `--color-bg` | `#FAFAF5` | `#0F1117` | Page background |
| `--color-surface` | `#FFFFFF` | `#1A1D27` | Card / content areas |
| `--color-text` | `#1A1A2E` | `#E8E8F0` | Body text |
| `--color-primary` | `#1B3A5C` | `#4A90D9` | Navy — headings, links |
| `--color-accent` | `#C8912A` | `#E8B84B` | Gold — CTAs, highlights |
| `--color-muted` | `#6B7280` | `#9CA3AF` | Dates, captions |

### Layout
- Mobile-first, single column
- Desktop: max-width container `1200px`, two-column grids for listing pages
- Sticky header (scrolls away on mobile, always-visible on desktop)
